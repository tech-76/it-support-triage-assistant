# Roadmap

## Current version: v2.1

The current version is a working service-desk triage assistant with structured support logic, sample tickets, bulk CSV analysis, and downloadable Markdown reports.

## Completed

- Working Gradio app
- Single-ticket triage
- Auto category detection
- Manual category override
- Priority assignment
- Escalation detection
- Likely cause generation
- Troubleshooting step generation
- Technician notes
- User-friendly response
- Recommended next action
- Confidence score
- Markdown report download
- Bulk CSV upload
- Sample tickets
- Privacy warning
- GitHub-ready documentation package

## v2.2: Presentation and usability improvements

Planned improvements:

## v5.0: Integrations

Possible integrations:

## v6.0: AI-assisted reasoning layer

Possible improvements:

- Add optional LLM support
- Keep rule-based triage as a safety layer
- Use environment variables for API keys
- Add prompt templates for category-specific troubleshooting
- Add redaction checks for sensitive information
- Add confidence and uncertainty explanations

## Long-term service idea

The project can become a small-business support intake tool that helps organize requests before a technician begins troubleshooting.

Potential paid versions:

- Setup fee for a branded intake assistant
- Monthly support workflow add-on
- Private client deployment
- Internal tool for freelance IT support work
- White-label intake assistant for small teams

## Important principle

Keep the project focused.

The strongest positioning is:

> Turn IT support requests into structured troubleshooting reports.

Avoid turning it into a generic chatbot too early.
